

服务端响应给前台时遵守以下数据结构
响应给前台格式为json格式
json中使用的引号一定为双引号

{
"code":"0000",
"msg":"create account ok",
"data":
    {
        "sessionId":"9915-6631-8871-0001",
        "accountName":"yellowuncle",
        "isActive":true
    }
}

服务器响应信息说明

code
响应编码4位字符串，注意是字符串不是数字。服务器任何一次响应时，响应编码必须存在。
0000 成功
9999 服务器异常

msg
响应消息为字符串类型。服务器任何一次响应时，响应消息必须存在。

data
响应数据为字符串或数组或json对象，由实际使用场景来决定。服务器任何一次响应时，响应数据可以不存在，可以不存在，可以不存在。
