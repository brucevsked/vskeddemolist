ASR录音机

以下描述了如何使用灵云ASR录音机功能。

1. ASR录音机简介

ASR录音机封装了系统的录音接口及灵云sdk的ASR接口，其目标是将输入的语音信号转换为相应的文本或命令。

2. ASR能力

ASR录音机封装了灵云SDK的ASR能力，其许多概念仍然是基于ASR基础API的。建议在使用灵云ASR录音机之前， 先熟悉 ASR能力 的相关内容。

在使用灵云ASR录音机之前，需要选择合适的ASR模块和能力以及音频编解码库， 具体请参见 选择模块和能力 和 音频编解码库。

3. 初始化

在使用录音机之前，仍然需要先初始化系统模块，请参见 系统模块的初始化/终止。

3.1 创建录音机对象

在初始化系统模块之后，就可以直接创建录音机对象：

// 创建录音机对象
ASRRecorder asrRecorder = new ASRRecorder();

在灵云SDK的Java版本中，在Windows/Linux平台下的ASR录音机和Android平台下的ASR录音机处于两个不同的包(package)中， 分别是 com.sinovoice.hcicloudsdk.pc.asr.recorder 和 com.sinovoice.hcicloudsdk.android.asr.recorder， 但都叫 ASRRecorder，二者的接口也是完全一致的，因此除了引入的包不一样外，其它代码都可以共享。

3.2 设置初始化参数

然后需要初始化录音机对象，主要是设置本地资源库的路径等：

// 创始初始化参数辅助类对象
AsrInitParam asrInitParam = new AsrInitParam();
// 如果使用本地能力，需要设置希望使用的能力，SDK利用此信息初始化模型库
asrInitParam.addParam(AsrInitParam.PARAM_KEY_INIT_CAP_KEYS, "asr.local.grammar");
// 如果使用本地能力，需要设置ASR语言模型文件的路径
String asrDataPath = "/mnt/sdcard/asrtest/";
asrInitParam.addParam(AsrInitParam.PARAM_KEY_DATA_PATH, asrDataPath);
// 设置获取本地文件名的特殊标记为none（不作特殊处理）
asrInitParam.addParam(AsrInitParam.PARAM_KEY_FILE_FLAG,"none");
// 用指定的初始化参数对录音机对象进行初始化
asrRecorder.init(asrInitParam.getStringConfig(), new ASRResultProcess());

如果是只使用云端能力，可以直接指定初始化配置串为空或者NULL。初始化配置串 pszConfig 的设置请参见 初始化能力。

注解
使用ASR录音机模块，并不需要调用ASR基础API的初始化操作，也即无需调用 HciCloudAsr.hciAsrInit()。

3.3 录音机监听器

为了能接收录音机内部信息，如状态信息、识别情况及错误信息等，应用层需要实现录音机监听器。下面的代码示例实现了一个监听器：

// 实现录音机监听器
private class ASRResultProcess implements ASRRecorderListener{
// 录音机状态变化事件回调
@Override
public void onRecorderEventStateChange(RecorderEvent recorderEvent) {
if(recorderEvent == RecorderEvent.RECORDER_EVENT_BEGIN_RECORD){
Log.v("recorder","开始录音");
} else if (recorderEvent == RecorderEvent.RECORDER_EVENT_BEGIN_RECOGNIZE){
Log.v("recorder","开始识别");
} else if (recorderEvent == RecorderEvent.RECORDER_EVENT_NO_VOICE_INPUT){
Log.v("recorder","无音频输入");
}
}
// 录音机识别完成事件回调
@Override
public void onRecorderEventRecogFinsh(RecorderEvent recorderEvent,
AsrRecogResult asrRecogResult) {
if(asrRecogResult!=null){
String sResult = asrRecogResult.getRecogItemList().get(0).getRecogResult();
Log.e("recorder", "识别结果为：" + sResult);
}
}
// 出错回调，通知外部出现的错误
@Override
public void onRecorderEventError(RecorderEvent recorderEvent, int errorCode) {
Log.e("recorder", "错误码为：" + errorCode);
}
// 录音回调，通知外部当前的录音数据和音量大小
@Override
public void onRecorderRecording(byte[] voiceData, int volumn) {
}
}

4. 启动录音机

使用start()来启动录音机，启动后录音机就自动开始了录音、端点检测、识别的过程。 以进行自由说识别为例：

// 使用自由说识别
String sConfig = "capKey=asr.cloud.freetalk,audioformat=pcm16k16bit");
// 自由说时无需指定语法文件
String sGrammarData = null;
// 开始识别
asrRecorder.start(sConfig, sGrammarData);

具体识别过程的概念、参数设置和注意事项请参见和 语法识别 和 自由说识别。

这里的配置项pszConfig包括了初始化会话的配置项（capKey和realtime）、识别选项、端点检测选项以及录音设备ID（deviceId）选项。

  • capKey选项请参见 ASR能力描述 。
  • 语法选项设置请参照ASR基础API的 3.6 语法识别的选项 或 4.2 自由说识别的选项 。
  • 端点检测选项设置请参照ASR基础API的 6.2 端点检测的选项 。
  • deviceId选项用于指定录音设备的ID，不指定则为系统自动匹配（WAVE_MAPPER） 。

当使用本地语法识别，可以把语法数据通过 sGrammarData 直接传入 start()中，并指明grammarType的配置项为wordlist或者jsgf。 但这种方式每次识别的时候都会重新加载语法，识别完后卸载语法，因此效率低下，不推荐使用。 本地语法识别时建议先使用 AsrRecorder.loadGrammar() 方式加载语法，获得语法ID， 在 AsrRecorder.start() 的时候指明grammarType为id来进行识别。示例如下：

// 以加载一个词表为例
String sWordList = "播放\r\n暂停\r\n停止\r\n\打开\r\n下一首\r\n上一首\r\n";
// 加载词表语法，获得语法id
AsrGrammarId grammarId;
asrRecorder.loadGrammar("grammarType=wordlist", sGrammarData, grammarId);
// 指定grammarType为id, grammarId为获得的语法id
String sRecogConfig = "capKey=asr.local.grammar,audioFormat=pcm16k16bit,encode=none,grammarType=id,grammarId=" + grammarId.getGrammarId();
// 无需指定语法数据
String sGrammarData = null;
asrRecorder.start(sRecogConfig, sGrammarData);
// 可以使用此id进行多次识别
// 不再使用此id时，进行卸载
asrRecorder.unloadGrammar(nGrammarId);

5. 端点检测

端点检测功能可以在传入的音频中检测到开始说话和停止说话的位置。录音机内部实现了端点检测能力， 无须开发者调用，但可以在调用 start()时的配置项中设置相关参数，相关参数请参见ASR基础API的 6.2 端点检测的选项 。

6. 结束录音

  1. 调用 start() 后录音机会自动进行端点检测，检测到末端后会自动启动识别。通过监听器的onRecorderEventRecogFinsh()返回结果后，自动停止。
  2. 通过 stopAndRecog(), 也可以不等端点检测检测到末端，直接结束录音，启动识别。通过监听器的onRecorderEventRecogFinsh()返回结果后，自动停止。
  3. 在录音过程中可以随时使用 cancel() 来结束录音或取消识别。

录音结束后，可以再使用 start() 函数进行下一轮的录音识别工作。

7. 获取录音机状态

可以使用 getRecorderState() 来获取录音机的当前状态：

int recorderState = asrRecorder.getRecorderState();
switch(recorderState)
{
// 不同状态的处理
...
}

8. 反初始化

当不再使用ASR录音机的时候，需要反初始化ASR录音机，示例如下：

// 反初始化
asrRecorder.release();

当不再使用其它灵云能力时，需要反初始化系统模块，请参见 系统模块的初始化/终止。

9. 注意

当前版本的播放器尚不支持 实时识别 (也即realtime选项)功能，以后的版本会提供。
